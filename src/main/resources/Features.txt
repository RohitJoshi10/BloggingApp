### 🤨 Iski zaroorat kyun hai?
1. **Fake Accounts:** Log fake emails se hazaron accounts bana lete hain jo tere database ko bhar denge.
2. **Security:** Agar kal ko user password bhool jaye (jo hum abhi kar rahe hain), toh hume kaise pata chalega ki ye sahi banda hai? Jab uska number verified hoga, tabhi toh hum use OTP bhej paayenge.
3. **Data Quality:** Tere paas sirf genuine users ka data rahega.

---

### 🛠️ Ye kaam kaise karta hai? (The Logic)

Verification do tarike se hoti hai jo tune abhi seekhi hain:

1. **OTP via SMS (Fast2SMS):** Register karte hi user ko SMS jayega. Jab tak wo OTP nahi daalega, uska account "Inactive" rahega.
2. **Verification Link via Email (SMTP):** Register karte hi ek link jayega (e.g., `api/v1/auth/verify?token=abc`). Link pe click karte hi account active ho jayega.




---

### 💻 Tere Code mein kya badlav (Changes) honge?

Agar tu ye implement karna chahta hai, toh ye 3 bade steps honge:

**1. Entity mein naya Column:**
`User` entity mein ek boolean field aayegi:
```java
private boolean isEnabled = false; // Default false rahega
```

**2. Registration Logic Update:**
`registerNewUser` mein ab hum user ko `save` toh karenge, par sath mein ek OTP/Token bhi bhejenge aur account ko `enabled = false` rakhenge.

**3. Login Logic Update:**
Jab user login karne ki koshish karega, toh hum check karenge:
* Agar `isEnabled == true` -> Login successful.
* Agar `isEnabled == false` -> Error denge: *"Bhai, pehle account verify karle!"*
